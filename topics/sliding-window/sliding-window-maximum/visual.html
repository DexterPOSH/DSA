Real-world analogy
Azure Stream Analytics computes a max over the active sliding window; a monotonic deque is the efficient local state for that. Evict stale events from the front, remove smaller candidates from the back, and read the max at the front.

Key Takeaway

Keep a decreasing deque of indices. Before adding, pop smaller values off the back (they can never be max again); drop the front if it slid out of the window. The front is the window max in O(1) — each index is pushed and popped once, so O(n) total.
