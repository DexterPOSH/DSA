Real-world analogy
Azure Resource Manager hierarchy can be scanned tier by tier. For each tier, the right-side view records the last scope seen at that level, just like BFS keeps the final node popped for each level.
Step description
Press "Next". BFS scans each level left-to-right; the last node popped per level is what you see from the right.

Key Takeaway

It's level-order BFS with one tweak: when i == len-1 within a level, that node is the rightmost — push it to the view. Note level 2's pick comes from the left subtree. O(n) time, O(n) space.
