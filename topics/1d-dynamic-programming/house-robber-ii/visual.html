House Robber II — split a circular Azure maintenance calendar

Recurring Azure maintenance calendar nums = [2, 7, 9, 3, 1] — first and last slots conflict.

🔁 Real-world analogy

Azure Update Manager on a recurring calendar has first and last maintenance windows adjacent. Break the circle into two cached linear schedules—exclude first or exclude last—and take the better result.

def rob(nums): if len(nums) == 1: return nums[0] def rob_line(houses): rob2, rob1 = 0, 0 for n in houses: rob2, rob1 = rob1, max(rob1, n + rob2) return rob1 return max(rob_line(nums[:-1]), # exclude last rob_line(nums[1:])) # exclude first

💡 Key takeaway

Break the circular Azure maintenance constraint into two linear cached schedules: one allows the first slot and excludes the last, the other allows the last slot and excludes the first. The first and last slots never appear together, so take the larger cached result. O(n) time, O(1) space.
