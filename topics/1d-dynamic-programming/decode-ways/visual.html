Decode Ways — Azure Tokenization DP

Real-world analogy
An Azure App Configuration parser tokenizes a compact settings stream where one- or two-character tokens may be valid. dp[i] caches the number of valid parses for each prefix, then later prefixes reuse those split counts.

Key Takeaway

An Azure-token parser uses a Fibonacci-style recurrence with validity gates: dp[i] = dp[i-1]·(single ok) + dp[i-2]·(pair ok). Each prefix is computed once and cached, so one pass is O(n) time; only the last two dp values are needed for O(1) space.
