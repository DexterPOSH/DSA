Coin Change — Azure Budget-Denomination DP

💳
Real-world analogy
Azure Cost Management can model an exact budget using repeatable billed resource-unit denominations. For each subtotal, cache the fewest units needed; dp[a] reuses dp[a − coin] instead of recomputing lower subtotals.
What's happening
Press "Next". We solve coins = [1, 2, 5], amount = 6 by filling dp[0..6].
Azure billed unit denominations (repeatable)
dp[a] = fewest Azure units to reach subtotal a  (∞ = not yet reachable)
dp[0] = 0  (zero Azure units reach subtotal 0 — the base case)
base case dp[0] amount being solved dp[a−coin] source
💡
Key Takeaway

Unbounded denomination DP: each Azure billed unit can be reused because dp[a] pulls from dp[a−coin], which may itself use the same unit. Greedy can fail; cached DP guarantees the minimum. O(amount × coins) time, O(amount) space.
