Minimum Interval to Include Each Query — offline sweep + size heap

For each query point, find the smallest interval covering it. Sort queries, add reachable intervals to a size-min-heap, lazily drop ended ones.

🛏️ The smallest-blanket analogy

Har query ek point hai; har interval ek blanket. Queries ko chote→bade process karo. Jaise point aage badhe, jin blankets ka left ≤ q ho gaya unhe size-heap me daalo. Jo blankets point se pehle khatam ho gaye (right < q) unhe top se nikaalo. Bacha sabse chhota blanket = answer.

Key takeaway

Process queries in sorted order so each interval is pushed once; a size-min-heap gives the smallest candidate, and lazy deletion of right < q entries keeps it valid. O((n+m) log n) total.
