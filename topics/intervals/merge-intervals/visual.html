Merge Intervals — Azure maintenance consolidation

Combine overlapping Azure maintenance windows into the fewest continuous windows. The whole trick is sorting by start first.

🛠️ Azure Update Manager analogy

Multiple teams may submit Azure maintenance windows for the same VM fleet. Sort the windows by start time, then sweep left to right: overlapping windows are consolidated by extending the end to max, while a gap starts a separate announced window.

Key takeaway

After sorting Azure maintenance windows by start, compare each window only with the last consolidated block. Overlap → extend the end with max; gap → announce a fresh window. O(n log n).
