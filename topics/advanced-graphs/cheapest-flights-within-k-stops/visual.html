🌐 The Azure hop-budget analogy

In Azure Traffic Manager, each region-to-region link has a latency or cost. Bellman-Ford rounds act like allowing one more hop in the route budget, while the snapshot prevents a candidate from using more than the allowed k+1 links.

Key takeaway

The max-hop limit maps to K+1 edges and K+1 relaxation rounds. Updating from a per-round snapshot guarantees each route gains exactly one hop per round, so a cheaper-but-too-long route cannot sneak in.
