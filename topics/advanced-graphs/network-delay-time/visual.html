🌐 The Azure Front Door analogy

Azure Front Door uses health and latency data to steer traffic toward the fastest reachable backend region. Dijkstra maps that network: each edge is latency, the min-heap commits the currently lowest-latency region, and the final delay is the slowest committed reachability time.

Key takeaway

The heap always hands you the lowest-latency unfinished node — so the first time a node is popped, its distance is final. Network delay = the largest of those final distances across the Azure-style topology.
