Number of 1 Bits — Azure Permission Flag Popcount

Real-world analogy
An Azure Service Bus SAS policy can be represented as a compact permission mask, with set bits for enabled rights. Each n & (n-1) pass clears the lowest enabled right, so the loop count equals the number of active permissions.

Key Takeaway

For an Azure permission mask, n & (n - 1) clears exactly one enabled right each time. Loop until n = 0 and the iteration count is the popcount — work is proportional to the number of active flags, not all 32 bit positions.
