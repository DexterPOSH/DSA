Longest Common Subsequence — filling the 2-D table

Longest sequence of characters appearing in both strings, in order (gaps allowed).

🎵 Analogy: comparing two playlists

Dono playlists fixed order me. Current gaane same? Match — dono aage badho, count +1 (diagonal). Same nahi? Kisi ek playlist ka gaana skip karo — jisme aage lambi match mile (top ya left ka max). Table me yaad rakho taaki dobara na sochna pade.

Problem. text1 = "AGCAT", text2 = "GAC". Longest common subsequence length? → Answer: 2 (e.g. "GC" or "AC")
match (text1[i-1]==text2[j-1]):  dp[i][j] = dp[i-1][j-1] + 1   (diagonal)
no match:  dp[i][j] = max( dp[i-1][j] , dp[i][j-1] )   (top / left)
computing now diagonal (match +1) top / left (max) filled
Press Next. Row 0 and column 0 are 0 — an empty prefix shares nothing.

Key takeaway

On a character match take the diagonal + 1; on a mismatch take the best of top/left. The bottom-right cell is the LCS length — built in O(m·n).
