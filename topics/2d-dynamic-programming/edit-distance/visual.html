Real-world analogy

Tum ek autocorrect engine ho: "horse" ko "ros" me badalna hai. Har insert / delete / replace ka cost 1 — sabse sasta raasta dhoondho. Table har prefix-pair ka cheapest answer cache kar leti hai.

Problem. Convert word1 = "horse" into word2 = "ros". → Minimum edits = 3  (replace h→r, delete r, delete e).
What's happening
Press Next to fill the table cell by cell. Rows = horse, columns = ros.
dp[i][j] = min edits to convert word1[:i] → word2[:j]
Where does this cell's value come from?

↑ Up · delete

💡
Key Takeaway

Each cell asks the same question on smaller prefixes: match → free diagonal, else 1 + cheapest of {replace ↖, delete ↑, insert ←}. Fill the whole (m+1)×(n+1) grid in O(m·n); the bottom-right cell is the answer.
