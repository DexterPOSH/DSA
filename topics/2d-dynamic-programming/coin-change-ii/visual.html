Coin Change II — counting combinations in a DP table

How many distinct ways to make an amount? Order doesn't matter — process coins one row at a time.

🪙 Analogy: a cashier counting ways to make change

Pehle ₹1 ke saare combos, phir ₹2 add karke, phir ₹5. Ek coin ko poora process karke hi agli pe jaate hain — isi "coins outer" order se 2+1+1+1 aur 1+2+1+1 same gini jaati hain (no double-count).

Problem. amount = 5, coins = [1, 2, 5]. Number of combinations? → Answer: 4
dp[c][a] = dp[c-1][a]  (skip this coin)  +  dp[c][a − coin]  (use it, reuse allowed)
computing now skip → cell above use → cell coin to the left, same row
Press Next. Column a=0 is all 1 — exactly one way to make 0 (take nothing). Row "∅" (no coins) is 0 for every positive amount.

Key takeaway

Each cell = skip the coin (cell above) + use the coin (cell coin steps left, same row). Looping coins outer, amount inner counts each combination once — the bottom-right cell is the answer.
