Interleaving String — weaving two strings

Can s3 be formed by interleaving s1 and s2, keeping each one's order?

⌨️ Analogy: two typists, one screen

Do typists ek shared screen pe type karte hain — ek apne s1 ke letters order me, dusra s2 ke. Screen pe bana combined text s3 hai. Har grid cell dp[i][j] poochta hai: kya s3 ke pehle i+j letters, s1 ke pehle i aur s2 ke pehle j letters se woven ho sakte hain?

Problem. s1 = "abc", s2 = "aed", s3 = "aabecd". Interleaving valid? → Answer: true
s3 = "aabecd"  (highlighted = the char being placed at the current cell)
dp[i][j] = ( s1[i-1]==s3[i+j-1] AND dp[i-1][j] )  OR  ( s2[j-1]==s3[i+j-1] AND dp[i][j-1] )
i.e. the last char came from s1 (cell above) or s2 (cell left)
computing now from s1 (cell above) from s2 (cell left)
Press Next. dp[0][0] = T — using zero chars from each, the empty s3 prefix is trivially an interleaving.

Key takeaway

Each cell asks "could the last placed char have come from s1 (above) or s2 (left)?" — true if that neighbor was reachable AND the char matches. The bottom-right cell answers the whole question in O(m·n).
