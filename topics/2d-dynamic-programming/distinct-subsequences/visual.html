Distinct Subsequences — counting the ways

How many distinct subsequences of s equal the target t?

🧾 Analogy: Azure Blob versioned policy snippet

Azure Blob Storage versioning can compare a long config blob against a target policy snippet. A match lets the cell count both use from the diagonal and skip from above; a mismatch only inherits skip.

Problem. s = "babb", t = "ab". Kitne distinct subsequences of s equal t? → Answer: 2
s[i-1] == t[j-1] :  dp[i][j] = dp[i-1][j] (skip s char) + dp[i-1][j-1] (use s char)
s[i-1] != t[j-1] :  dp[i][j] = dp[i-1][j]   (can only skip)
computing now skip s-char (cell above) use s-char (diagonal)
Press Next. Column 0 is all 1 — there is exactly one way to form the empty t (highlight nothing). Row 0 (beyond col 0) is 0 — empty s can't form a non-empty t.

Key takeaway

On a character match a cell adds the "use" (diagonal) and "skip" (above) counts; on a mismatch it just inherits "skip". The bottom-right cell counts all distinct subsequences in O(m·n) — here 2.
