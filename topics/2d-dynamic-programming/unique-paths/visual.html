Unique Paths — filling the DP grid cell by cell

A robot moves only right or down. Count distinct paths from top-left to bottom-right.

🤖 Analogy: one-way city streets

Har intersection tak pahunchne ke tareeke = (upar wale corner ke tareeke) + (left wale corner ke tareeke). Robot sirf wahin se aa sakta hai. Bas har cell ko apne top + left ka sum bana do.

Problem. Grid m = 3 rows × n = 4 cols. Count paths from (0,0) to (2,3). → Answer: 10
dp[i][j] = dp[i-1][j]  (top)  +  dp[i][j-1]  (left)
base (=1) computing now top + left it reads filled
Press Next to start. First we seed the top row and left column with 1 — only one straight-line path reaches those.

Key takeaway

Each cell is just top + left. Filling the grid row-by-row turns an exponential path-count into an O(m·n) sweep — the bottom-right cell holds the answer.
