Target Sum — counting sign assignments

Assign + or − to every number; count the ways to hit the target.

🪙 Analogy: two pockets

Har coin ko left pocket (+) ya right pocket (−) me daalo. End me running sum exactly target chahiye. Alag sign-choices jab same running sum pe milein, aage ka kaam identical — to unhe ek count me merge karo. Yahi 2-D DP: rows = "kitne numbers process kiye", columns = "running sum abhi kya hai".

Problem. nums = [1, 1, 1, 1, 1], target = 3. Kitne sign-assignments sum ko 3 banate hain? → Answer: 5
dp[i][s] = number of ways to reach running sum s using first i numbers
dp[i][s] = dp[i-1][s − nums[i-1]] (chose +)  +  dp[i-1][s + nums[i-1]] (chose −)
cell computing now came from (+ / −) filled target column (s=3)
Press Next. Row 0 = "0 numbers used": exactly 1 way to be at running sum 0 (do nothing), all other sums have 0 ways.

Key takeaway

Each new number shifts every reachable sum up by +num and down by −num, adding the two source counts together. Read dp[n][target] — here 5 — for the answer in O(n·S).
