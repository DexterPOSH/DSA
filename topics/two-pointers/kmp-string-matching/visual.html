KMP String Matching — the Smart Slide

Knuth–Morris–Pratt finds a pattern P inside a text T in O(n + m) by never re-reading a text character. The trick is a precomputed LPS table (Longest proper Prefix that is also a Suffix). Two linked animations below: first build LPS, then search.

Real-world analogy — Azure WAF signature scanning

🛡️

Azure Web Application Firewall scans a request body for a signature using a fallback table: after a partial match fails, it reuses the longest matched prefix that is also a suffix. The payload pointer keeps moving forward, just like KMP's text pointer.

Why you can't hand-roll a single pass without the LPS table

  • The seductive bug. A natural-looking "one loop, advance j on match, reset j=0 on mismatch" passes easy tests but is silently wrong on inputs where a partial match overlaps the pattern's own prefix.
  • Adjacent matches — searching P="ab" in T="abab". After matching index 0–1 you must be ready to match again at index 2. Resetting j=0 on the next char works here, but the same naive reset breaks the next case…
  • Overlapping prefix — P="aab" in T="aaab". You match a,a then hit a≠b at text index 2. Naive code resets j=0 and consumes that a as "no match" — but it was actually the start of a fresh aa! You'd miss the match at index 1. LPS says j = lps[1] = 1, keeping one matched a so the next char completes aab. Correct.
  • The fix is the table. On mismatch with j>0, do j = lps[j-1] (fall back, don't advance text i); only when j==0 do you advance i. That single line is the whole algorithm — and it's impossible to get right without precomputing LPS.
O(n · m)

Naive re-scan. On every mismatch the text pointer backtracks to start+1 and you recompare from scratch. Worst case (e.g. T="aaaa…a", P="aaa…b") does ~n·m comparisons.

O(n + m)

KMP. Build LPS in O(m), then scan T once. Text pointer i never moves backward; j only falls back via the table. Total comparisons are linear — provably no re-reads.

Key takeaway

Build a fallback table once (length = lps[length-1] on prefix mismatch), then search with a text pointer that only ever moves forward: match → advance both; mismatch with j>0 → j = lps[j-1] (slide, reusing the matched prefix); mismatch with j==0 → advance i. The LPS table is what makes the slide correct.
