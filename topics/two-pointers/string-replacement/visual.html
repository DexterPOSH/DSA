String Replacement — Scan & Swap

Implement replace(orig, find, repl) by hand — no str.replace, no regex. Walk a single pointer across the string, try to match find at each spot, and build a fresh result.

Real-world analogy

You're a proofreader sweeping a finger across a manuscript looking for the phrase "Fox" to swap with "Cat". At each letter you peek ahead: do the next few letters spell "Fox"? If yes, you write "Cat" onto a clean sheet and skip your finger past the whole phrase. If no, you copy the single letter under your finger and nudge forward by one. The clean sheet you build up is the answer.

Edge cases to defend in the interview

  • Empty find — there is no substring to locate; an empty needle "matches everywhere." Treat as an error / throw (or define it as a no-op) rather than looping forever.
  • find == repl — swapping a phrase for itself changes nothing. Return orig unchanged (and skip the work).
  • find not present — every position mismatches, so you just copy orig char for char. Result equals the input.
  • Overlapping matches — only non-overlapping occurrences count. With find="aa" on "aaaa" you jump by len(find) after each hit, giving 2 matches, not 3. (Replacing left-to-right and skipping the consumed window enforces this.)
  • null / empty inputs — orig="" returns ""; null orig → return null or throw per contract; null repl often means delete (treat as ""). Decide and state the contract.
  • Case sensitivity — "fox" ≠ "Fox". Matching is case-sensitive by default; mention a caseInsensitive flag (compare lower-cased) only if asked.
O(n · m)

Naive scan (this walkthrough). For each of the n start positions you may compare up to m = len(find) chars before a mismatch. Simple, and usually plenty for interview inputs.

O(n + m)

KMP. Precompute a failure (LPS) table over find in O(m), then scan orig once without ever re-examining a char — no backtracking on the text. Mention it as the optimal follow-up.

Key takeaway

One pointer, two moves: on a full match emit repl and jump i += len(find); on a mismatch copy orig[i] and do i += 1. Building a fresh result string — instead of mutating in place — keeps replacements clean and naturally non-overlapping.
