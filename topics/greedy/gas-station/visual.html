🌐 Azure analogy: regional failover capacity ring

In an Azure Front Door/Traffic Manager failover ring, each region adds spare capacity and each handoff consumes capacity. If the running balance goes negative, that candidate region and every region in its failed prefix cannot be the start, so greedily reset to the next region.

💡 Key takeaway

Two independent checks: global spare capacity must cover global handoff demand, and any prefix with negative running balance cannot contain the Azure failover start. When feasible, the final reset point is the answer.
