🔎 Analogy: Azure Cosmos DB range index seek

An Azure Cosmos DB range index keeps keys ordered, so a point lookup probes the middle of the remaining index range. Each comparison discards the lower or upper half, matching binary search on a sorted array.

Problem. Sorted array nums = [-1, 0, 3, 5, 9, 12], find target = 9. → return index 4.
searching for 9

Key takeaway

On sorted data, compare mid with the target and keep only the half that can still contain it — O(log n), O(1) space.
