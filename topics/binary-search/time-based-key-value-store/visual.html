Time Based Key Value Store — floor via binary search

For get(key, t) we want the value at the largest timestamp ≤ t. Timestamps are sorted, so we binary-search for the "floor".

🕒 Analogy: Azure Blob Storage versioning

With Azure Blob Storage versioning, an as-of read at time t returns the newest blob version whose timestamp is ≤ t. The timestamp list is sorted, so binary search saves each valid version and moves right to find the latest still-valid one.

Key takeaway

When ts[mid] ≤ target, that value is a valid answer — save it to res and search right for an even larger-but-still-valid timestamp. When ts[mid] > target, go left. The last saved res is the floor. O(log n) per query.
