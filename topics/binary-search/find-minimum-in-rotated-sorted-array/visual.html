Find Minimum in Rotated Sorted Array — which half contains the wrap point?

Compare nums[mid] with nums[hi] to decide which half to keep.

🔄 Analogy: Azure Cosmos DB partition ring wrap

An Azure Cosmos DB partition key range map is ordered around a hash ring, but a rotated view has one wrap point from high hashes back to low hashes. Comparing nums[mid] with nums[hi] tells which half can still contain that wrap, so the search halves the ring each step.

Problem. Rotated sorted array nums = [4, 5, 6, 7, 0, 1, 2]. Find the minimum. → 0 (at index 4).

Purple cell = nums[hi], our reference for "is this half sorted?"

Key takeaway

If nums[mid] > nums[hi], the wrap point and minimum are to the right, so set lo = mid + 1; otherwise the minimum is at mid or to its left, so set hi = mid.
