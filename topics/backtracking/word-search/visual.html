Real-world analogy

Trace a word in a newspaper puzzle with a pencil. Mark each cell used as you step onto it. Hit a dead end? Lift the pencil, un-mark, and try another direction. That undo is backtracking.

Key takeaway

DFS extends the path one adjacent cell at a time, matching word[i]. A cell is marked # while it's on the current path so it can't be reused; on a dead end we restore it and back off to try a different neighbor.
