Array Equal Partition — K Equal-Sum Subsets

LeetCode 698. Backtracking se array ko k buckets mein baanto, har bucket ka sum exactly target ho. Pruning ke saath exponential search tractable ban jaati hai.

Real-world analogy

Socho k = 4 dost restaurant mein hain aur bill ko exactly equally split karna hai — har dost ko same amount dena hai (yahan amount = target = total / k). Items (dishes) ek-ek karke uthate ho aur har item ko kisi ek dost ke "share" mein daalte ho.

Agar kisi dost ka share target se zyada ho jaye → woh combination galat hai, item wapas uthao (backtrack) aur dusre dost ki taraf try karo. Agar saare items baant diye aur sab shares barabar nikle → bill perfectly split ho gaya.

Pruning rules (yahi tractability deti hain)

  • Capacity prune: agar bucket + val > target → us bucket mein daalo hi mat, skip.
  • Duplicate empty buckets: agar current bucket empty tha aur usme place karne se aage fail hua, toh baaki empty buckets try mat karo — woh identical hain, same fail dega. (Yahan se sabse bada saving aata hai.)
  • Sorted descending: bade elements pehle → branching factor girta hai, dead-ends jaldi.
  • Exact-fit shortcut: agar bucket + val == target, woh bucket "full" lock ho jaata hai aur seedha agle element pe recurse.
  • Early reject: koi single val > target → immediately false.

Complexity

Brute force: O(k · 2n)

Har element ke liye k buckets ka choice → search space exponential. n elements, har ek k options.

Pruning ke saath: practically tractable
  • Duplicate-empty-bucket prune symmetric branches kaat deta hai.
  • Descending sort + capacity prune dead-ends jaldi expose karta hai.
  • Space: O(k + n) — buckets array + recursion depth.
  • Worst case still exponential, par real inputs pe yeh fast nikalta hai.

Key takeaway

Har element ko har bucket mein "try → recurse → backtrack" karo. Teen pruning — capacity overflow skip, identical-empty-bucket skip, aur descending sort — naive O(k·2n) ko real inputs pe tezi se solvable bana dete hain.
