Kth Largest in a Stream — Min-Heap of Size k

Real-world analogy

Ek VIP lounge me sirf top-3 spenders ki seats hain. Naya customer aata hai — agar uska bill lounge ke sabse chhote member se bada hai, andar aa jaata hai aur sabse chhota nikal diya jaata hai. Kth largest ka jawab? Lounge ka sabse chhota member — wahi min-heap ka root hai.

k = 3, initial nums = [4, 5, 8, 2], then stream add(3), add(5), add(10), add(9), add(4). Har add ke baad 3rd largest return karo.

Key takeaway

Keep only the top-k elements in a min-heap. Its root is always the kth largest. Each add is just push + maybe pop — O(log k), no full sort, no growing memory.
