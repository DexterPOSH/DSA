Kth Largest in an Array — Min-Heap of Size k

Real-world analogy

Tumhe class ka 4th-highest scorer chahiye, poori class rank kiye bina. Ek chhoti leaderboard of size k rakho jisme sirf top-k scorers hain. Naya score top-k me ghuse to leaderboard ka lowest gir jaata hai. Antt me leaderboard ka sabse neeche wala = kth largest. Wahi min-heap ka root hai.

nums = [3, 2, 1, 5, 6, 4], k = 2. Maintain a min-heap of size 2 (the 2 largest seen). Root is the 2nd largest. Answer: 5.

Alternative — Quickselect, O(n) average: partition around a random pivot so it lands in its final sorted slot. If that slot is index n − k, it's the answer; otherwise recurse into just one half. No heap, in-place — but worst case O(n²).
Key takeaway

Keep a min-heap of size k: a value enters only if it beats the root (the weakest of the current top-k). When the scan ends, the root is the kth largest — found in O(n log k), never sorting the whole array.
