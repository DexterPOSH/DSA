Design Twitter — Event Hubs-style k-way merge with a max-heap

Each followed source is an ordered event stream; use a heap to merge current heads into the newest-first feed.

📡 Analogy: Azure Event Hubs partitions

Azure Event Hubs keeps each partition in order. Treat each user feed as one ordered partition: put each current head event in a max-heap by timestamp, emit the newest head, then advance only that partition. That is the same heap-based k-way merge used to build the feed.

Key takeaway

Like merging Azure Event Hubs partitions, a max-heap holding one current head per source emits the global newest-first feed while touching only the events you actually return.
