Longest Consecutive Sequence — Set + Run Starts

🎟️
Real-world analogy
Random numbered tickets ko ek table pe daal do (hash set) taaki ek nazar me pata chale koi number hai ya nahi. Chain sirf un tickets se shuru karo jinka ek kam wala (n−1) maujood hi nahi — woh run-start hai. Phir n, n+1, n+2… count karo. Har number ek hi baar visit → O(n).
Step description

💡
Key Takeaway

Dump everything into a set, then only extend runs from their start (a number whose n−1 is absent). That boundary check is what keeps each element visited once, turning a naive O(n²) into a clean O(n).
